# Contributing to Confounder

First off, thank you for considering contributing to Confounder! It's people like you that make open source such a great community.

## 1. Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issues](https://github.com/your-username/confounder/issues) first to see if someone else has already created a ticket. If not, go ahead and make one!

## 2. Fork & create a branch

If this is something you think you can fix, then fork Confounder and create a branch with a descriptive name.

## 3. Implementation Guidelines

- **Style**: We use `ruff` to enforce PEP8 standards.
- **Testing**: We use `pytest`. All new features and bug fixes must include tests. 
- **Type Hints**: Please use Python 3.10+ type hints. MyPy is configured to strictly check typing.

## 4. Make a Pull Request

At this point, you should switch back to your master branch and make sure it's up to date with Confounder's master branch.

Once it's up to date, you can submit your Pull Request. Please follow the instructions in the Pull Request template.
